Runtime 底层 之 isMemberOfClass && isKindOfClass的探究

2020/03/06 posted in  Runtime

isMemberOfClass和isKindOfClass一直以来都是想当然的使用,有时候得不到想要的结果,最后决定探究一下,彻底搞清楚,闲话不多说,先看下源码;

isMemberOfClass

+ (BOOL)isMemberOfClass:(Class)cls {
    return object_getClass((id)self) == cls;
}
- (BOOL)isMemberOfClass:(Class)cls {
    return [self class] == cls;
}

isMemberOfClass有两个方法,一个是实例方法,一个是类方法,他俩的区别是:

  • 实例方法- (BOOL)isMemberOfClass:(Class)cls: 根据实例对象取得类对象,再去判断
  • 类方法 + (BOOL)isMemberOfClass:(Class)cls:根据对象取得元类对象,再去判断

其实简单的说是根据传入的类的 isa,来判断,因为实例对象的 isa 是实例的类对象,类对象的 isa 指向的是类的元类,Meta Class

验证:
创建两个类,Person 类继承自 NSObject
Student 类继承自 Person

先看代码,思考一下打印结果是什么

res1 : [Person class] Person 类的元类 是不是和[NSObject class]类一致? 肯定是不一致 所以返回 NO
res2: [Person class] Person 类的元类 是不是和[Person class]类一致? 元类和类肯定也是不一致 所以返回 NO

实例对象;

第一行,Person new 出来一个实例,该实例的 isa 指向类 Person, 而 Person 很明显不是 NSObject类,所以为 0;
第二行,Person new 出来一个实例,该实例的 isa 指向类 Person, 判断 Person 类是不是 Person,答案是肯定的,所以为 1;

object_getClass(cls) 这个方法返回是传入类的元类

isKindOfClass

  • 调用者是传入的类的实例对象，或者调用者是传入类的继承者链中的类的实例对象，则返回YES
  • 判断调用者是否是传入对象的子类，别弄反了
  • 去父类递归查找判断
+ (BOOL)isKindOfClass:(Class)cls {
    for (Class tcls = object_getClass((id)self); tcls; tcls = tcls->super_class) {
        if(tcls == cls) return YES;
    }
    return NO;
}
-（BOOL)isKindOfClass:(Class)cls {
    for(Class tcls = [self class]; tcls; tcls = tcls->super_class) {
        if(tcls == cls) return YES;
    }
    return NO;
}
