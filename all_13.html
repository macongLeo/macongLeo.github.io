类 子类 分类 区别与联系

分类(category)
分类允许开发者在不改变原有类的情况下,对该类进行拓展,分类是 OC 的特有语法,他是表示一个指向分类的结构体指针,原则上分类只能增加方法,不能增加实例变量,因为在分类中,类的实例变量布局已经固定,所以不能添加实例变量,但是可以利用 runtime 的特性,添加关联属性.

Category
Category 是表示一个指向分类的结构体的指针，其定义如下：
typedef struct objc_category *Category;
struct objc_category {
  char *category_name                          OBJC2_UNAVAILABLE; // 分类名
  char *class_name                             OBJC2_UNAVAILABLE; // 分类所属的类名
  struct objc_method_list *instance_methods    OBJC2_UNAVAILABLE; // 实例方法列表
  struct objc_method_list *class_methods       OBJC2_UNAVAILABLE; // 类方法列表
  struct objc_protocol_list *protocols         OBJC2_UNAVAILABLE; // 分类所实现的协议列表
}

从上面可以看出,这个结构体主要包含了分类定义的实例方法与类方法,但是这里面并没有属性列表,也就没办法直接添加属性(实例变量)
要点:

  • 分类是用于给原有类添加方法的,因为在分类的结构体中只有方法列表,没有属性列表.
  • 分类中可以写@property,但是不会生成 getter/setter 方法,也不会生成实例变量.编译器会有警告,但是不会报错
  • 可以在分类中访问原有类的.h 中的属性
  • 如果分类中有和原有类同名的方法,会优先调用分类中的方法,也就是会忽略原有类的方法,同名方法优先级"分类>类>父类",所以分类中不要和类中原有方法同名
  • 如果多个分类中都有和元类同名的方法,那么调用该方法的时候,由编译器决定,编译器会执行最后一个参与编译的分类中的方法.

拓展
主类,子类,分类中+load方法的执行顺序

  • +load 方法是在 mian 函数执行之前执行的,并不会主动调用
  • 主类,分类,子类都会调用+load 方法
  • 调用顺序:主类的+load方法会先调用,然后调用子类的+load,最后调用分类的+load.
  • 所有分类的+load方法都会调用,调用顺序取决于它的编译顺序.

+ Initialize方法的调用顺序

  • + Initialize只有当真正用到的时候,并且重写了+ Initialize方法才会调用
  • 当调用子类的+ Initialize方法的时候,如果父类重写了+ Initialize方法,会先调用父类的+ Initialize,然后调用子类的+ Initialize方法,如果此时父类的分类也重写了+ Initialize方法,那么父类的+ Initialize方法就会被覆盖掉
